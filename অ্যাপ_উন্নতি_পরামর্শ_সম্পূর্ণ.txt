সমস্যা ১: API Error প্রথম বারে আসে, দ্বিতীয়বার কাজ করে
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

মূল কারণ (Root Cause):

  বর্তমান কোডে (gemini.ts) কোনো AUTO-RETRY নেই।
  প্রথমবার যদি:
  • নেটওয়ার্ক সংযোগ ধীর হয় (45s timeout হিট করে)
  • Gemini API cold start করে (নতুন session-এ প্রথম call)
  • rateLimiter.waitForSlot() delay দেয়
  • Google API 503/500 ক্ষণিকের জন্য দেয়

  ...তাহলে সরাসরি error throw হয়, retry হয় না।
  দ্বিতীয়বার চাপলে API warm থাকে → কাজ করে।

সমাধান — Auto-Retry Logic:

  gemini.ts-এ callGeminiStream ও callGemini ফাংশনের
  ভেতরে retry wrapper যুক্ত করতে হবে।

  ─── এই কোড pattern অন্য AI-কে দিয়ে implement করান ───

  // gemini.ts-এর শুরুতে যোগ করুন (existing functions-এর আগে):

  const MAX_AUTO_RETRIES = 2;
  const RETRY_DELAY_MS = [2000, 5000]; // 2s তারপর 5s

  async function withAutoRetry<T>(
    fn: () => Promise<T>,
    attempt: number = 0
  ): Promise<T> {
    try {
      return await fn();
    } catch (e: any) {
      const isRetryable =
        !e.message?.includes('429') &&
        !e.message?.includes('quota') &&
        !e.message?.includes('Quota') &&
        attempt < MAX_AUTO_RETRIES;

      if (isRetryable) {
        console.warn(`[Gemini] Retry ${attempt + 1}/${MAX_AUTO_RETRIES} after ${RETRY_DELAY_MS[attempt]}ms`);
        await new Promise(r => setTimeout(r, RETRY_DELAY_MS[attempt]));
        return withAutoRetry(fn, attempt + 1);
      }
      throw e;
    }
  }

  ─── WriteScreen-এ user-friendly error message ───

  বর্তমানে catch block-এ শুধু একটি alert দেওয়া হয়।
  পরিবর্তন করুন:

  } catch (e: any) {
    let msg = e.message || String(e);
    const isNetworkError = msg.includes('fetch') || msg.includes('Failed to fetch') || msg.includes('abort');
    const isQuotaError = msg.includes('Quota') || msg.includes('quota') || msg.includes('429');

    if (isQuotaError) {
      msg = '⚠️ API Quota শেষ। অন্য API key ব্যবহার করুন বা কিছুক্ষণ পরে চেষ্টা করুন।';
    } else if (isNetworkError) {
      msg = '🌐 নেটওয়ার্ক সমস্যা। ইন্টারনেট সংযোগ পরীক্ষা করুন এবং আবার চেষ্টা করুন।';
    } else {
      msg = `API Error: ${msg}\n\n💡 আবার Generate বাটন চাপুন — সাধারণত দ্বিতীয়বারে কাজ হয়।`;
    }
    displayAlert(msg);
  }

সমস্যা ৪: AI কি ১০ পাতার Document লিখতে পারে?
         Chunked Generation কীভাবে করবেন?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

বর্তমান অবস্থা বিশ্লেষণ:
  • maxOut: 16384 tokens → প্রায় 12,000 শব্দ
  • ১০ পাতার document ≈ 3,000-4,000 শব্দ → এর মধ্যেই পড়ে
  • তাই সাধারণত একটানা generate করতে পারে
  • কিন্তু যদি truncate হয়, "Continue Generating" বাটন আছে

সমস্যা কোথায়:
  ১. "Continue" বাটন manual — user-কে ক্লিক করতে হয়
  ২. Free tier: gemini-2.5-flash = 10 RPM, 1500 RPD (requests/day)
  ৩. বড় document-এর prompt নিজেই বড় হলে input tokens বেশি খরচ হয়

সমাধান — Auto-Continue (Chunked Generation):

  WriteScreen.tsx-এ handleGenerate এর পরে এই logic যোগ করুন:

  // Auto-continue যদি truncated হয়
  const handleGenerateWithAutoComplete = async () => {
    await handleGenerate(); // existing function call

    // After initial generation, check if truncated
    // Maximum 3 auto-continuations to avoid infinite loop
    let continueCount = 0;
    while (isTruncated && continueCount < 3) {
      await new Promise(r => setTimeout(r, 1500)); // 1.5s gap (rate limit safety)
      await handleContinueGenerating(); // existing function
      continueCount++;
    }
  };

  ─── Rate Limit Safe Chunked Prompt ───

  বড় document-এর জন্য section-by-section approach:

  const sections = [
    'Preamble and Brief Facts only',
    'Submissions of the Noticee',
    'Discussion and Issue Analysis',
    'Findings and ORDER'
  ];

  // প্রতি section আলাদাভাবে generate করুন
  // এতে প্রতিটি call ছোট হবে, rate limit কম লাগবে
  // তারপর সব combine করে একসাথে দেখান

  ─── Free Tier Optimization ───

  বর্তমান rate limiter: 10 RPM (প্রতি মিনিটে ১০ call)
  Practical suggestion:
  • Format mode: দ্রুত, কম tokens → fine
  • AI mode: বেশি tokens → প্রতি generate-এ ১টি call → fine
  • Legal mode (নতুন): বড় document → auto-continue হলে ৩-৪ call
    → Rate limiter automatically wait করবে

  কোনো পরিবর্তন দরকার নেই রাতলিমিটারে।
  শুধু UI-তে দেখান: "Generating... (2/4 sections)"
